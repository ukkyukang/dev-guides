# 기여 가이드 (Contributing Guide)

Dev Guides에 강의를 추가하거나 수정하고 싶다면 아래 가이드라인을 따라주세요.

## 새 강의 추가하기

### 1. 디렉토리 생성

다음 번호를 사용하여 디렉토리를 생성합니다:

```bash
mkdir -p NN-강의이름/{assets,examples}
```

예시:
```bash
mkdir -p 08-kubernetes/{assets,examples}
```

### 2. README.md 작성

아래 템플릿을 기반으로 강의 문서를 작성합니다:

```markdown
# NN. 강의 제목

> 한 줄 요약 설명

## 📋 사전 요구사항
## 🎯 학습 목표
## 📖 본문
## 🏗️ 예제 프로젝트
## 🔗 참고 자료
## ⏭️ 다음 강의
```

### 3. 예제 프로젝트 추가

- `examples/` 디렉토리 아래에 번호와 이름으로 예제를 생성합니다
- 각 예제는 **독립 실행 가능**해야 합니다
- 필요한 설정 파일(`pyproject.toml`, `Dockerfile` 등)을 모두 포함해주세요

### 4. 루트 README.md 업데이트

`README.md`의 강의 목차 테이블에 새 강의를 추가합니다.

## 명명 규칙

- **디렉토리**: `NN-소문자-하이픈-구분` (예: `08-kubernetes`)
- **예제**: `NN-소문자-하이픈-구분` (예: `01-basic-setup`)
- **에셋**: `소문자-하이픈-구분.확장자` (예: `architecture-diagram.png`)

## assets/ 디렉토리 구성 가이드

`assets/` 폴더에는 강의 문서를 보충하는 다음 자료들을 보관합니다:
- **이미지**: 아키텍처 다이어그램, 터미널 실행 결과 스크린샷, 로고 등
- **다이어그램 소스**: Mermaid 소스 파일이나 드로잉 도구의 작업 파일
- **보조 자료**: 요약 PDF, 치트 시트, 짧은 데모 GIF 등

## 강의 순서 변경

강의 사이에 새 강의를 삽입해야 하는 경우, 기존 디렉토리의 번호를 재조정합니다.
디렉토리 이름 변경 후 모든 크로스 링크를 업데이트해야 합니다.
